# HospitalSorting

## Работа по курсу "Рефакторинг баз данных и приложений"

### Выбран превый сценарий "Проектирование и разработка новой программной системы"

### Описание проекта
> Идея проекта основана на реальной проблеме и предложена старшим врачом скорой помощи университетской клиники ПСПбГМУ

При поступлении больного в стационар сразу ставится задача определения тяжести его состояния, так называемый триаж. Задача по своей сути строго алгоритмическая и заключается в сопоставлении параметров и симптомов больного с одним из трех типов состояний (легкое, среднее, тяжелое). Данное приложение будет состоять из двух клиентских и одного серверного модуля. Разница между клиентами будет в том, что один будет реализовывать функционал для записи параметров больного медсестрой, а другой будет предназначен для врача скоро помощи.

## Клиент (медсестра)

Веб приложение со следующим функционалом:
- Добавление карточки больного
- Заполнение данных о больном
- Возможность корректировки данных о больном

## Клиент (врач)

Веб приложение со следующим функционалом:
- Демонстрация карточки больных с выбранным состоянием
- Коррекция состояния тяжести больного в БД по воле врача

## Сервер

 - Restful сервис поддерживающий обработку соотвествующих клиентам запросов
 - БД для сохранения данных больных и служебной информации (врачи, медсестры)
